thrift.transport.memory

class TMemoryBuffer : TBaseTransport [final] ¶
A transport that simply reads from and writes to an in-memory buffer. Every time you call write on it, the data is simply placed into a buffer, and every time you call read, data is consumed from that buffer.
Currently, the storage for written data is never reclaimed, even if the buffer contents have already been read out again.
this() ¶
Constructs a new memory transport with an empty internal buffer.
this(size_t capacity) ¶
Constructs a new memory transport with an empty internal buffer, reserving space for capacity bytes in advance.
If the amount of data which will be written to the buffer is already known on construction, this can better performance over the default constructor because reallocations can be avoided.

If the preallocated buffer is exhausted, data can still be written to the transport, but reallocations will happen.

capacitySize of the initially reserved buffer (in bytes).

Returns a read-only view of the current buffer contents.

Note:

For performance reasons, the returned slice is only valid for the life of this object, and may be invalidated on the next write() call at will – you might want to immediately .dup it if you intend to keep it around.
